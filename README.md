# StepTrace: Android 센서 기반 만보기 및 위치 기록 앱

## 프로젝트 개요

StepTrace는 Android 디바이스의 내장 센서를 활용하여 사용자의 걸음 수를 측정하고, 일정 걸음 수마다 위치 정보를 기록하는 만보기 앱 프로토타입이다.

단순히 버튼 입력이나 수동 기록으로 걸음 수를 관리하는 것이 아니라, Android의 `TYPE_STEP_COUNTER` 센서를 활용해 실제 디바이스 움직임 기반으로 걸음 수를 측정하는 구조를 구현했다.
또한 걸음 수가 일정 기준에 도달했을 때 GPS 위치 정보를 함께 저장하여, 사용자의 이동 기록을 확장할 수 있는 구조를 실험했다.

* 실시간 걸음 수 측정
* 걸음 수 화면 출력
* Activity 생명주기에 따른 센서 리스너 등록 및 해제

* GPS 기반 마지막 위치 정보 조회
* 위도/경도 데이터를 JSON 형태로 저장하는 구조 설계

## 구현 내용

### 1. Step Counter 센서 연동

Android의 `SensorManager`를 사용하여 디바이스 내장 Step Counter 센서를 가져오고, `SensorEventListener`를 통해 센서 값 변화를 감지하도록 구현했다.

앱 실행 중 센서 값이 변경되면 현재 누적 걸음 수를 읽고, 화면의 TextView에 표시하는 구조를 구성했다.

```text id="tv8zlm"
SensorManager 초기화
→ TYPE_STEP_COUNTER 센서 조회
→ SensorEventListener 등록
→ 센서 값 변경 감지
→ 걸음 수 화면 표시
```

---

### 2. Activity 생명주기 기반 센서 관리

앱이 화면에 표시되는 동안에만 센서 값을 수신하도록 `onResume()`에서 센서 리스너를 등록하고, 앱이 일시정지되면 `onPause()`에서 리스너를 해제하는 구조를 적용했다.

이를 통해 불필요한 센서 사용을 줄이고, Android Activity 생명주기에 맞춰 센서 자원을 관리하는 방식을 경험했다.

---

### 3. 걸음 수 기반 위치 기록 구조

걸음 수가 100 단위에 도달했을 때 위치 정보를 가져와 기록하는 구조를 설계했다.

```text id="jz0spy"
걸음 수 측정
→ 걸음 수가 100 단위인지 확인
→ 위치 정보 조회
→ 위도/경도 JSON 생성
→ 이동 기록 배열에 저장
```

이 구조를 통해 단순 만보기 기능을 넘어, 사용자의 이동 구간 또는 활동 이력을 위치 정보와 결합할 수 있는 가능성을 실험했다.

---

### 4. GPS 위치 정보 조회

`LocationManager`를 사용하여 GPS Provider의 마지막 위치 정보를 조회하고, 위도와 경도 값을 JSON 객체에 저장하는 구조를 구현했다.

위치 권한이 없는 경우 `ACCESS_FINE_LOCATION` 권한을 요청하도록 처리하여, Android 위치 권한 처리 흐름도 함께 다뤘다.

## 기술적으로 의미 있었던 점

이 프로젝트에서는 Android 디바이스의 하드웨어 센서를 직접 활용하는 경험을 했다.
특히 Step Counter 센서를 통해 걸음 수 데이터를 읽고, Activity 생명주기에 맞춰 센서 리스너를 관리하는 구조를 구현했다.

또한 걸음 수 데이터와 위치 데이터를 결합하여 단순 표시 앱이 아니라 이동 기록 기반 앱으로 확장할 수 있는 가능성을 검토했다.
이를 통해 모바일 앱에서 센서 데이터, 위치 정보, JSON 기반 기록 구조를 함께 다루는 기본 흐름을 경험했다.

## 한계 및 개선점

이 프로젝트는 Android 센서 활용을 실험한 초기 프로토타입이므로, 실제 서비스 수준으로 확장하기 위해서는 다음 개선이 필요하다.

* 위치 권한 처리 안정화
* GPS 위치 객체 null 처리
* JSON 객체 초기화 로직 보완
* 걸음 수 초기값 보정
* 앱 재실행 시 데이터 저장 및 복원
* Room DB 또는 SQLite 기반 로컬 저장 구조 추가
* 백그라운드 걸음 수 측정 지원
* 이동 경로 시각화 기능 추가
* 배터리 사용량 최적화
